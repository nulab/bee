:::note[前提]
これらのプロンプトは [using-bee スキル](../integrations/ai-agent/)がインストールされている環境で使うことを想定しています。スキルが bee のコマンド体系や `--json` オプションなどの基本パターンを AI に教えるため、プロンプトでは「何をしたいか」の指示に集中できます。
:::

## 課題の棚卸し

<Tabs>
<TabItem label="期限切れの課題を一覧">

```
MY_PROJECT の未完了かつ期限切れの課題を一覧にして。

bee issue list -p MY_PROJECT --json key,summary,dueDate,assignee,status で取得して、
dueDate が今日より前かつ status.name が「完了」でないものだけフィルタして、
期限が古い順に表形式で出力して。
```

</TabItem>
<TabItem label="担当者別の課題数を集計">

```
MY_PROJECT の未完了課題を担当者別に集計して。

bee issue list -p MY_PROJECT --json assignee,status で全件取得して、
status.name が「完了」でないものを担当者ごとにカウントして、
件数が多い順に表形式でまとめて。未アサインは「未割り当て」として集計して。
```

</TabItem>
<TabItem label="放置されている課題を検出">

```
MY_PROJECT で放置されている課題を探して。

bee issue list -p MY_PROJECT --json key,summary,status,assignee,updated で取得して、
status.name が「処理中」かつ updated が2週間以上前のものをリストアップして。
担当者ごとにグループ化して表示して。
```

</TabItem>
</Tabs>

## 課題の一括操作

<Tabs>
<TabItem label="マイルストーンの一括変更">

```
MY_PROJECT でマイルストーン「v1.0」に紐づいている未完了課題を、
すべてマイルストーン「v1.1」に変更して。

1. bee milestone list -p MY_PROJECT --json で両方のマイルストーン ID を確認
2. bee issue list -p MY_PROJECT --json key,milestone,status で
   milestone が「v1.0」かつ未完了の課題を特定
3. 対象の各課題に対して bee issue edit <課題キー> --milestone <v1.1のID> を実行
4. 変更した課題の一覧を表示して
```

</TabItem>
<TabItem label="カテゴリの一括付与">

```
MY_PROJECT で「バグ」種別の未完了課題のうち、カテゴリが未設定のものに
「要トリアージ」カテゴリを付与して。

1. bee category list -p MY_PROJECT --json で「要トリアージ」カテゴリの ID を確認
2. bee issue-type list -p MY_PROJECT --json で「バグ」種別の ID を確認
3. bee issue list -p MY_PROJECT --json key,issueType,category,status で
   対象の課題を特定（issueType が「バグ」、category が空、未完了）
4. 各課題に bee issue edit <課題キー> --category <カテゴリID> を実行
5. 変更結果をまとめて表示して
```

</TabItem>
</Tabs>

## リリース準備

<Tabs>
<TabItem label="リリースノートの下書き">

```
MY_PROJECT のマイルストーン「v1.2」に含まれる完了済み課題からリリースノートを作って。

1. bee milestone list -p MY_PROJECT --json でマイルストーン ID を確認
2. bee issue list -p MY_PROJECT --json key,summary,issueType,category で
   該当マイルストーンの完了済み課題を取得
3. issueType.name で以下のセクションに分類して Markdown で出力して:
   - ✨ 新機能（種別が「タスク」「要望」）
   - 🐛 バグ修正（種別が「バグ」）
   - 🔧 改善（それ以外）
4. 各項目は「- [課題キー] 課題タイトル」の形式にして
```

</TabItem>
<TabItem label="マイルストーンの未完了チェック">

```
MY_PROJECT のマイルストーン「v1.2」のリリース準備状況を確認して。

1. bee milestone list -p MY_PROJECT --json で期限と ID を確認
2. bee issue list -p MY_PROJECT --json key,summary,status,assignee,priority で
   該当マイルストーンの全課題を取得
3. 以下をまとめて:
   - ステータスごとの件数（完了/処理中/未対応など）
   - 未完了課題を優先度の高い順にリストアップ（担当者付き）
   - リリース可否の判断材料として、ブロッカーになりそうな課題があれば指摘して
```

</TabItem>
</Tabs>

## ドキュメント・Wiki

<Tabs>
<TabItem label="議事録のテンプレ作成">

```
MY_PROJECT の Backlog ドキュメントに今日の日付で議事録を作成して。

bee document create -p MY_PROJECT --title "議事録 YYYY-MM-DD" で作成して。
--body には以下のテンプレートを Markdown で入れて:

## 日時
YYYY-MM-DD HH:MM

## 参加者
-

## アジェンダ
1.

## 決定事項
-

## TODO
- [ ]
```

</TabItem>
<TabItem label="Wiki の棚卸し">

```
MY_PROJECT の Wiki で、長期間更新されていないページをリストアップして。

bee wiki list -p MY_PROJECT --json name,updated で全ページを取得して、
updated が6ヶ月以上前のものだけフィルタして、
古い順に表形式（ページ名・最終更新日）で出力して。
```

</TabItem>
</Tabs>

## プロジェクト横断

<Tabs>
<TabItem label="自分の全課題を一覧">

```
自分にアサインされている全プロジェクトの未完了課題をまとめて。

1. bee project list --json で参加プロジェクト一覧を取得
2. 各プロジェクトに対して bee issue list -p <プロジェクトキー> -a @me --json key,summary,dueDate,status,priority で未完了課題を取得
3. 全プロジェクトの結果をマージして、期限が近い順にソート
4. プロジェクトキー・課題キー・タイトル・期限・優先度を表形式でまとめて
5. 期限切れの課題があれば先頭に分けて表示して
```

</TabItem>
<TabItem label="通知の一括既読">

```
未読通知を整理して。

1. bee notification list --json で未読通知を取得
2. 1週間以上前の通知を特定して件数を教えて
3. 確認を取ってから、該当する通知を bee notification read で1件ずつ既読にして
4. 最後に bee notification count で残りの未読数を表示して
```

</TabItem>
</Tabs>
