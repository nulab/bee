import { Tabs, TabItem } from "@astrojs/starlight/components";

:::tip[using-bee スキルで短いプロンプトでOK]
[using-bee スキル](../integrations/ai-agent/)をインストールすると、AI エージェントが bee のコマンド体系・`--json` オプション・`@me` ショートハンドなどを自動的に把握します。そのため、プロンプトでは **bee コマンドの使い方を細かく指示する必要はありません** — やりたいことを自然言語で伝えるだけで、エージェントが適切なコマンドを選んで実行します。

プロンプトに **「Backlog」** というキーワードを含めると、スキルが自動的にトリガーされます。以下のプロンプト例はすべてこのキーワードを含んでいます。
:::

:::note
プロンプト中の `MY_PROJECT` や課題キーは、実際のプロジェクトキーや課題キーに読み替えてください。
:::

## 課題の棚卸し

<Tabs>
<TabItem label="期限切れの課題を一覧">

```
Backlog の MY_PROJECT で、未完了かつ期限切れの課題を期限が古い順に表にして。
```

</TabItem>
<TabItem label="担当者別の課題数を集計">

```
Backlog の MY_PROJECT で、未完了課題を担当者別に集計して件数が多い順に表にして。
未アサインも「未割り当て」として含めて。
```

</TabItem>
<TabItem label="放置されている課題を検出">

```
Backlog の MY_PROJECT で、「処理中」のまま2週間以上更新されていない課題を探して。
担当者ごとにグループ化してリストアップして。
```

</TabItem>
</Tabs>

## 課題の一括操作

<Tabs>
<TabItem label="マイルストーンの一括変更">

```
Backlog の MY_PROJECT で、マイルストーン「v1.0」に紐づいている未完了課題を
すべて「v1.1」に移動して。変更した課題の一覧も見せて。
```

</TabItem>
<TabItem label="カテゴリの一括付与">

```
Backlog の MY_PROJECT で、種別が「バグ」かつカテゴリ未設定の未完了課題に
「要トリアージ」カテゴリを付与して。変更結果もまとめて見せて。
```

</TabItem>
</Tabs>

## リリース準備

<Tabs>
<TabItem label="リリースノートの下書き">

```
Backlog の MY_PROJECT で、マイルストーン「v1.2」の完了済み課題からリリースノートを作って。
種別をもとに「新機能」「バグ修正」「改善」にカテゴリ分けして、
各課題のキーとタイトルを箇条書きにした Markdown 形式で。
```

</TabItem>
<TabItem label="マイルストーンの未完了チェック">

```
Backlog の MY_PROJECT で、マイルストーン「v1.2」のリリース準備状況を確認して。
ステータスごとの件数と、未完了課題を優先度の高い順にリストアップして。
ブロッカーになりそうな課題があれば指摘して。
```

</TabItem>
</Tabs>

## ドキュメント・Wiki

<Tabs>
<TabItem label="議事録のテンプレ作成">

```
Backlog の MY_PROJECT に、今日の日付で議事録ドキュメントを作成して。
テンプレートは「日時・参加者・アジェンダ・決定事項・TODO」のセクション構成で。
```

</TabItem>
<TabItem label="Wiki の棚卸し">

```
Backlog の MY_PROJECT で、6ヶ月以上更新されていない Wiki ページを古い順にリストアップして。
```

</TabItem>
</Tabs>

## プロジェクト横断

<Tabs>
<TabItem label="自分の全課題を一覧">

```
Backlog の全プロジェクトから、自分にアサインされている未完了課題をまとめて。
期限が近い順にソートして、期限切れのものは先頭に分けて表にして。
```

</TabItem>
<TabItem label="通知の一括既読">

```
Backlog の未読通知を確認して、1週間以上前のものは既読にして。
対象件数を教えてから実行して、最後に残りの未読数も表示して。
```

</TabItem>
</Tabs>
