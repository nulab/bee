---
title: GitHub Issues のように PR と課題を連動させる
description: GitHub Issues の「Closes #123」のように、PR の操作で Backlog 課題のステータス更新・クローズを自動化する
---

GitHub Issues では、PR に `Closes #123` と書くだけで Issue のリンク・ステータス更新・クローズが自動で行われます。このレシピで Backlog 課題でも同じ操作感を実現できます。

| GitHub Issues での体験 | このレシピで実現すること |
|---|---|
| PR を作ると Issue にリンクが付く | 課題に PR リンクをコメント＋ステータスを「処理中」に |
| PR の本文で Issue を追加・削除できる | 新たに追加された課題キーにだけコメント |
| PR をマージすると Issue がクローズされる | 課題を自動クローズ |

PR 本文に `PROJECT-123` のような課題キーを書くだけで動作します。複数の課題キーにも対応しています。

:::note[前提]
リポジトリの **Settings → Secrets and variables → Actions** に `BACKLOG_API_KEY` と `BACKLOG_SPACE` を登録してください。設定方法は [CI/CD での利用](../integrations/ci-cd/) を参照してください。
:::

```yaml
name: Sync PR Lifecycle to Backlog Issues
on:
  pull_request:
    types: [opened, edited, closed]
    branches: [main]

jobs:
  sync:
    runs-on: ubuntu-latest
    env:
      BACKLOG_API_KEY: ${{ secrets.BACKLOG_API_KEY }}
      BACKLOG_SPACE: ${{ secrets.BACKLOG_SPACE }}
    steps:
      - name: PR 本文から課題キーを収集する
        id: collect
        run: |
          KEYS=$(echo "$PR_BODY" | grep -oE '[A-Z_]+-[0-9]+' | sort -u | tr '\n' ' ')
          echo "検出した課題キー: ${KEYS:-(なし)}"
          echo "keys=$KEYS" >> "$GITHUB_OUTPUT"

          # edited イベント用: 変更前の本文からもキーを取得
          if [ "${{ github.event.action }}" = "edited" ]; then
            PREV=$(echo "$PREV_BODY" | grep -oE '[A-Z_]+-[0-9]+' | sort -u | tr '\n' ' ')
            echo "変更前の課題キー: ${PREV:-(なし)}"
            echo "prev=$PREV" >> "$GITHUB_OUTPUT"
          fi
        env:
          PR_BODY: ${{ github.event.pull_request.body }}
          PREV_BODY: ${{ github.event.changes.body.from }}

      # PR 作成 → 課題にリンク＋ステータスを「処理中」に
      - name: 各課題のステータスを「処理中」に変更して PR リンクをコメントする
        if: github.event.action == 'opened' && steps.collect.outputs.keys != ''
        run: |
          for KEY in ${{ steps.collect.outputs.keys }}; do
            echo "処理中に変更: $KEY"
            npx @nulab/bee issue edit "$KEY" --status processing \
              || echo "⚠ 失敗: $KEY のステータス変更に失敗しました"
            echo "コメント追加: $KEY に PR リンクを送信します"
            npx @nulab/bee issue comment "$KEY" \
              --body "レビュー中: ${{ github.event.pull_request.html_url }}" \
              || echo "⚠ 失敗: $KEY へのコメントに失敗しました"
          done

      # PR 本文の編集 → 新しく追加された課題キーにだけコメント
      - name: 新しく追加された課題に PR リンクをコメントする
        if: github.event.action == 'edited' && steps.collect.outputs.keys != ''
        run: |
          for KEY in ${{ steps.collect.outputs.keys }}; do
            if echo "${{ steps.collect.outputs.prev }}" | grep -qw "$KEY"; then
              echo "スキップ: $KEY (変更前から存在)"
              continue
            fi
            echo "コメント追加: $KEY に PR リンクを送信します"
            npx @nulab/bee issue comment "$KEY" \
              --body "レビュー中: ${{ github.event.pull_request.html_url }}" \
              || echo "⚠ 失敗: $KEY へのコメントに失敗しました"
          done

      # PR マージ → 課題をクローズ
      - name: 各課題をクローズする
        if: github.event.action == 'closed' && github.event.pull_request.merged == true && steps.collect.outputs.keys != ''
        run: |
          for KEY in ${{ steps.collect.outputs.keys }}; do
            echo "クローズ: $KEY"
            npx @nulab/bee issue close "$KEY" \
              --comment "PR #${{ github.event.pull_request.number }} のマージにより自動クローズ" \
              || echo "⚠ 失敗: $KEY のクローズに失敗しました"
          done
```

## カスタマイズ

- **ステータス変更が不要な場合** — `bee issue edit` の行を削除すれば、リンクのコメントだけになります。
- **自動クローズが不要な場合** — 「各課題をクローズする」ステップを削除してください。
- **クローズではなく別のステータスにしたい場合** — `bee issue close` を `bee issue edit --status processed` に変更してください。
- **`edited` イベントが不要な場合** — トリガーの `types` から `edited` を削除し、「新しく追加された課題に PR リンクをコメントする」ステップを削除してください。
