---
title: PR 作成時に課題へリンクを貼る
description: ブランチ名や PR 本文から Backlog 課題キーを抽出し、PR の URL を課題にコメントする
---

ブランチ名に `PROJECT-123` のような課題キーを含める運用をしていれば、PR 作成時に Backlog 課題へ自動でリンクを追加できます。PR 本文の編集で課題キーが追加・削除された場合にも対応します。

:::note[前提]
リポジトリの **Settings → Secrets and variables → Actions** に `BACKLOG_API_KEY` と `BACKLOG_SPACE` を登録してください。設定方法は [CI/CD での利用](../integrations/ci-cd/) を参照してください。
:::

```yaml
name: Link PR to Backlog Issue
on:
  pull_request:
    types: [opened, edited]

jobs:
  link:
    runs-on: ubuntu-latest
    env:
      BACKLOG_API_KEY: ${{ secrets.BACKLOG_API_KEY }}
      BACKLOG_SPACE: ${{ secrets.BACKLOG_SPACE }}
    steps:
      - name: ブランチ名と PR 本文から課題キーを収集する
        id: collect
        run: |
          BRANCH_KEYS=$(echo "${{ github.head_ref }}" | grep -oE '[A-Z_]+-[0-9]+' || true)
          BODY_KEYS=$(echo "$PR_BODY" | grep -oE '[A-Z_]+-[0-9]+' || true)
          CURRENT_KEYS=$(echo -e "${BRANCH_KEYS}\n${BODY_KEYS}" | sort -u | grep -v '^$' | tr '\n' ' ')
          echo "検出した課題キー: ${CURRENT_KEYS:-(なし)}"
          echo "current_keys=$CURRENT_KEYS" >> "$GITHUB_OUTPUT"

          if [ "${{ github.event.action }}" = "edited" ] && [ -n "$PREV_BODY" ]; then
            PREV_KEYS=$(echo "$PREV_BODY" | grep -oE '[A-Z_]+-[0-9]+' | sort -u | tr '\n' ' ')
            echo "変更前の課題キー: ${PREV_KEYS:-(なし)}"
          else
            PREV_KEYS=""
          fi
          echo "prev_keys=$PREV_KEYS" >> "$GITHUB_OUTPUT"
        env:
          PR_BODY: ${{ github.event.pull_request.body }}
          PREV_BODY: ${{ github.event.changes.body.from }}

      - name: 新しく追加された課題に PR リンクをコメントする
        if: steps.collect.outputs.current_keys != ''
        run: |
          CURRENT="${{ steps.collect.outputs.current_keys }}"
          PREV="${{ steps.collect.outputs.prev_keys }}"

          for KEY in $CURRENT; do
            if [ "${{ github.event.action }}" = "edited" ]; then
              if echo "$PREV" | grep -qw "$KEY"; then
                echo "スキップ: $KEY (変更前から存在)"
                continue
              fi
            fi
            echo "コメント追加: $KEY に PR リンクを送信します"
            npx @nulab/bee issue comment "$KEY" \
              --body "PR が作成されました: ${{ github.event.pull_request.html_url }}" \
              || echo "⚠ 失敗: $KEY へのコメントに失敗しました"
          done
```

## ポイント

- ブランチ名の規約は `feature/PROJECT-123-description` や `PROJECT-123/description` など自由に決められます。正規表現 `[A-Z_]+-[0-9]+` で最初にマッチした課題キーを使います。
- `edited` イベントでは `github.event.changes.body.from` で変更前の本文を取得できるため、新たに追加されたキーにだけコメントを送ります。すでにリンク済みの課題に重複コメントが飛ぶことはありません。
- PR 本文から課題キーが削除された場合、Backlog 側のコメントは残ります。コメントの自動削除が必要な場合は Backlog API で該当コメントを検索・削除するステップを追加してください。
