---
title: 期限切れ課題を定期リマインドする
description: GitHub Actions の schedule トリガーで、期限を過ぎた Backlog 課題に自動でリマインドコメントを追加する
---

GitHub の Stale bot に相当するパターンです。定期実行で期限切れの課題を検出し、リマインドコメントを追加します。

```yaml
name: Remind Overdue Backlog Issues
on:
  schedule:
    # 毎週月曜 9:00 JST (= 日曜 24:00 UTC)
    - cron: "0 0 * * 1"
  workflow_dispatch:

jobs:
  remind:
    runs-on: ubuntu-latest
    env:
      BACKLOG_API_KEY: ${{ secrets.BACKLOG_API_KEY }}
      BACKLOG_SPACE: ${{ secrets.BACKLOG_SPACE }}
    steps:
      - name: Find and remind overdue issues
        run: |
          TODAY=$(date +%Y-%m-%d)
          npx @nulab/bee issue list --project MY_PROJECT --json key,dueDate,status \
            | jq -r --arg today "$TODAY" \
                '.[] | select(.dueDate != null and .dueDate < $today and .status.name != "完了") | .key' \
            | while read -r KEY; do
                npx @nulab/bee issue comment "$KEY" \
                  --body "この課題は期限を過ぎています。ステータスを確認してください。" \
                  || echo "Warning: Failed to comment on $KEY"
              done
```

## ポイント

- `workflow_dispatch` も追加しておくと、手動実行でテストできます。
- `MY_PROJECT` はプロジェクトキーに置き換えてください。複数プロジェクトを対象にしたい場合は `for` ループで回せます。
- リマインド頻度やメッセージはチームの運用に合わせて調整してください。
