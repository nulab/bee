---
title: PR マージで課題を自動クローズする
description: GitHub の「Closes #123」に相当する、PR マージ時に Backlog 課題を自動クローズするワークフロー
---

GitHub の `Closes #123` に相当するパターンです。PR マージをトリガーに、ブランチ名または PR タイトルから課題キーを抽出してクローズします。

:::note[前提]
リポジトリの **Settings → Secrets and variables → Actions** に `BACKLOG_API_KEY` と `BACKLOG_SPACE` を登録してください。設定方法は [CI/CD での利用](../integrations/ci-cd/) を参照してください。
:::

```yaml
name: Close Backlog Issue on Merge
on:
  pull_request:
    types: [closed]
    branches: [main]

jobs:
  close-issue:
    if: github.event.pull_request.merged == true
    runs-on: ubuntu-latest
    env:
      BACKLOG_API_KEY: ${{ secrets.BACKLOG_API_KEY }}
      BACKLOG_SPACE: ${{ secrets.BACKLOG_SPACE }}
    steps:
      - name: Extract issue key
        id: extract
        run: |
          # PR タイトルを優先し、なければブランチ名から取得
          ISSUE_KEY=$(echo "${{ github.event.pull_request.title }}" | grep -oE '^[A-Z_]+-[0-9]+')
          if [ -z "$ISSUE_KEY" ]; then
            ISSUE_KEY=$(echo "${{ github.head_ref }}" | grep -oE '[A-Z_]+-[0-9]+' | head -1)
          fi
          echo "issue_key=$ISSUE_KEY" >> "$GITHUB_OUTPUT"

      - name: Close issue
        if: steps.extract.outputs.issue_key != ''
        run: |
          npx @nulab/bee issue close ${{ steps.extract.outputs.issue_key }} \
            --comment "PR #${{ github.event.pull_request.number }} のマージにより自動クローズ"
```

## ポイント

- `--comment` でクローズ理由を残せます。
- クローズではなくステータスを「処理済み」に変えたい場合は `bee issue close` を `bee issue edit --status processed` に変更してください。
